# 02. 데이터 전달(EventListener)

CollectorService가 파싱·정제한 센서 데이터를 RuleEngine으로 전달하는 구간입니다. CollectorService와 RuleEngine을 하나의 서버로 두기로 결정하였으므로, 별도의 메시지 브로커 없이 Spring의 `ApplicationEvent`와 `@Async`를 이용해 비동기로 전달합니다.

## 전달 흐름

### A. 센서 데이터
- MQTT 구독을 통해 센서로부터 데이터를 받습니다.

### B. 이벤트 발생
- 파싱·정제한 센서 데이터가 준비될 떄마다 던질 이벤트를 구성합니다.

```java
// 파싱·정제가 완료된 센서 데이터를 담아 발행할 이벤트
public class SensorDataReadyEvent {

    // ...
}
```

- `@Component`어노테이션을 사용하여, 이벤트를 처리할 리스너(`@EventListener`)들을 모아둔 빈을 준비합니다. `@Async`를 이용해 이벤트를 비동기로 전달합니다.

```java
// SensorDataReadyEvent를 수신하여 RuleEngine으로 넘기는 리스너
public class SensorDataReadyEventListener {

    // ...
}
```

1. 데이터 준비가 완료되면 `ApplicationEventPublisher`에게 이벤트를 던집니다.

```java
import org.springframework.context.ApplicationEventPublisher;

// 파싱·정제를 수행하고, 완료 시 SensorDataReadyEvent를 발행하는 서비스
public class CollectorService {

    // ...
    private final ApplicationEventPublisher eventPublisher;
    
    public void ingest(/*...*/) {
        
        // ...
        
        eventPublisher.publishEvent(new SensorDataReadyEvent(/*...*/));
    }
}
```

2. 등록된 이벤트 리스너 빈은 이벤트가 발생하였을 때, Spring에 의해 각각 올바른 이벤트 리스터로 유도됩니다.

### C. 데이터 전달
- 이로써 파싱·정제된 데이터는 비동기로 RuleEngine에게 전달됩니다.

```
[MQTT 구독으로 수신된 센서 데이터]
                ↓
        (파싱·정제 완료)
                ↓
   eventPublisher.publishEvent(...)
                ↓
   ─────────────────────────────
   (@Async, 호출자는 여기서 대기하지 않고 리턴)
   ─────────────────────────────
                ↓
   Spring이 이벤트 타입을 보고 해당 @EventListener를 자동 매칭
                ↓
        [SensorDataReadyEventListener]
                ↓
          ruleEngineFlow.start(...)
                ↓
           [RuleEngine 진입]
```

## D. `@Async` 설정

`@Async`가 실제로 별도 스레드에서 동작하려면, `@EnableAsync`와 스레드 풀(`Executor`) 설정이 필요합니다.

```java
@EnableAsync
@Configuration
public class AsyncConfig {
 
    @Bean(name = "taskExecutor")
    public Executor taskExecutor() {
        // ThreadPoolTaskExecutor 생성
        // corePoolSize, maxPoolSize, queueCapacity 등 설정
    }
}
```

`@EnableAsync`가 등록되어 있지 않으면 `@Async`는 조용히 무시되고 동기로 실행되므로, 반드시 함께 구성해야 합니다.
